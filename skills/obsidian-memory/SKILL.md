# Obsidian AI 记忆系统

一套以 Obsidian 为载体的 AI 外置记忆中枢，实现跨会话长期记忆、对话沉淀、项目追踪与知识网络互联。

## 核心能力

- **AI 长期记忆**：双层上下文（当前活跃 + 长期），每次对话开始自动加载
- **对话沉淀**：对话结束后提炼知识、错误、成功案例写入对应层级
- **项目追踪**：所有内容围绕项目主线清晰可追溯
- **知识网络**：笔记互联，拒绝孤岛
- **第一性原理学习**：基于 LLM Wiki × Karpathy 方法论的知识进化管道
- **日常事务追踪**：任务面板、待办跟进、会议结论存档
- **学习辅助**：收件箱捕获、第一性原理深挖、费曼输出

## 使用指南

### 在本 Vault 工作时的优先规则

当当前工作目录是用户的 Obsidian Vault 时，先把它当作第二大脑使用：

1. 读取 `01-AI核心层/AI-当前活跃上下文.md`
2. 读取 `01-AI核心层/AI-长期上下文.md`
3. 读取 `02-时间线层/每日日志/YYYY-MM-DD.md`（若当天日志不存在，按需创建）
4. 读取 `07-任务层/任务面板` 和 `06-经验层/检查清单` 中与任务相关的文件
5. 读取 `03-对话层/` 最新 1-3 个 session，理解近期讨论和决策
6. 对话产生重要结论、经验、知识、操作手册或调研结果时，判断完立即写入对应目录，不等待用户再次提醒

使用当前日期和用户所在时区创建日志/session。当前 Codex 环境中优先使用工作目录下的实际 Vault 路径，不要套用旧的 Windows 路径。

### 快速判断当前任务

1. **初次搭建** → 执行 [初始化流程](#初始化流程)
2. **加载上下文** → 执行 [加载记忆流程](#加载记忆流程)
3. **保存会话** → 执行 [保存会话流程](#保存会话流程)
4. **创建笔记** → 参考 [笔记创建规范](#笔记创建规范)
5. **更新 AI 上下文** → 执行 [更新上下文流程](#更新上下文流程)

### 第二步：创建 AI 核心层笔记

在 `01-AI核心层/` 下创建两个核心文件，参考 `references/templates.md` 中的 **AI 当前活跃上下文模板** 和 **AI 长期上下文模板**。

### 第三步：配置 Hooks（可选但推荐）

运行 `python3 scripts/setup_hooks.py <vault_path>` 生成以下 4 个 hook 脚本。默认输出到 `~/.openclaw/obsidian/hooks/`，也可以通过 `OBSIDIAN_HOOKS_DIR` 环境变量指定：

| 文件 | Hook 类型 | 用途 |
|------|-----------|------|
| `session_start_load.py` | SessionStart | 启动时加载 Obsidian 记忆 |
| `obsidian_session_load.py` | UserPromptSubmit | 每条消息注入 AI 上下文 |
| `session_end_summary.py` | SessionEnd | 对话结束保存摘要 |
| `save_session.py` | 手动执行 | 手动保存完整会话 |

并输出 `hooks.json` 配置示例，供用户合并到自己的 agent runtime 配置。

### 第四步：推荐安装 Obsidian 插件

| 插件 | 用途 | 推荐度 |
|------|------|--------|
| Templater | 动态模板（日期变量等） | ⭐⭐⭐⭐⭐ |

| Calendar | 日历视图 | ⭐⭐⭐⭐ |

## 加载记忆流程

当用户要求加载 Obsidian 上下文时：

1. 确认 vault 路径（优先从用户输入读取，也可从 `OBSIDIAN_CONFIG` 指向的 JSON 配置文件读取）
2. 读取 `01-AI核心层/AI-当前活跃上下文.md`
3. 读取 `01-AI核心层/AI-长期上下文.md`
4. 读取 `02-时间线层/每日日志/YYYY-MM-DD.md`
5. 读取 `03-对话层/` 中最新的 1-3 个 session 文件
6. 汇总输出，格式：

```
✅ 记忆已加载

📌 当前活跃上下文：[内容摘要]
🧠 长期上下文：[内容摘要]
💬 最近会话：[最新 session 标题]
```

---

## 保存会话流程

对话结束或用户要求保存时：

1. 提炼本次对话核心内容（50-200字）
2. 判断类型：有知识→知识卡片 | 有错误→错误记录 | 有成功→成功案例
3. 在 `03-对话层/` 创建或更新 `session-YYYY-MM-DD-主题.md`（使用 `references/templates.md` 中的会话记录模板）
4. 追加摘要到 `01-AI核心层/AI-当前活跃上下文.md` 的"最近关键结论"部分
5. 如有需要，同步创建对应的知识/经验笔记

## 每日工作流（自动触发）

### 早上进入（首次对话）
```
1. 读 01-AI核心层/AI-当前活跃上下文.md
2. 读 01-AI核心层/AI-长期上下文.md
3. 读 02-时间线层/每日日志/YYYY-MM-DD.md（没有则创建）

5. 读 06-经验层/检查清单
6. 读 03-对话层/ 最新 1-3 个 session
7. 检查 05-知识层/收件箱 有没有新碎片
8. 有更新 → 立即写回每日日志和对应 INDEX
```

### 对话结束后
```
1. 有重要结论 → 写 03-对话层/session-YYYY-MM-DD-主题.md

3. 对话中提到新概念 → 写 05-知识层/收件箱
4. 理解了底层原理 → 写 05-知识层/第一性原理
5. 能教会小白 → 写 05-知识层/费曼输出
6. 成功完成操作 → 写 05-知识层/操作手册
7. 有调研结论 → 写 05-知识层/研究笔记 或费曼输出
8. 遇到专业术语 → 写 05-知识层/术语表
9. 发现好工具/文章 → 写 05-知识层/资源资料
10. 有灵感/架构想法 → 写 08-灵感层
```

### 每周五
```
1. 写 02-时间线层/每周回顾/
2. 清理 05-知识层/收件箱 → 分流到各分类
3. 更新 07-任务层/任务面板
4. 盘点本月技术调研成果 → 写费曼输出或永久笔记
```

## 知识层工作流（核心学习系统）

本系统基于 **LLM Wiki × Karpathy × 第一性原理** 方法论构建：

```
收件箱（碎片捕获）
  ↓ 理解原理后写
第一性原理（探索"为什么"，底层机制）
  ↓ 提炼概念后写
概念（"是什么"，精确定义）
  ↓ 深挖原因后写
原理（"为什么"，因果关系）
  ↓ 用最简单的话写
费曼输出（讲给小白听，一句话懂）
  ↓ 整理归档
永久笔记（沉淀为可引用的知识卡片）
```

### 各目录写入时机

| 目录 | 写入时机 | 格式要求 |
|------|---------|---------|
| **收件箱** | 对话中提到新概念/看到好文章摘要 | 碎片文字 + 来源链接 |
| **第一性原理** | 理解了一个技术的底层"为什么" | 回答一个核心问题："XX是什么/为什么" |
| **概念** | 能精确提炼一个概念的定义 | 一句话定义 + 核心要点 + 链接 |
| **费曼输出** | 能用最简单的话教给小白 | 一句话定义 + 类比 + 不超过3个要点 + 例子 |
| **操作手册** | 成功完成某个操作时 | 步骤编号 + 注意事项 + 常见坑 |
| **永久笔记** | 收件箱清理时 | 概念 + 定义 + 至少3条双向链接 |
| **术语表** | 遇到新术语时 | 术语 + 定义 + 使用场景 |
| **原理** | 理解因果机制时 | "为什么"的因果链 + 关键公式 |
| **工作日志** | 日常工作随手记时 | 流水账 + 关键动作 + 结果 |
| **资源资料** | 发现好工具/文章时 | 工具名 + 链接 + 简短说明 |
| **研究笔记** | 做技术调研时 | 调研对象 + 关键结论 + 链接 |
| **从零构建** | 从头学一个领域时 | 领域概述 + 学习路径 + 关键概念 |

### 读取时机

| 目录 | 读取时机 |
|------|---------|
| 第一性原理 | 技术选型、问题排查、技术讨论前 |
| 费曼输出 | 需要给别人解释某个概念时 |
| 操作手册 | 重复操作时查这里 |
| 永久笔记 | 需要引用某个知识时 |
